Plan de trabajo para la administración de Linux en Parrot OS

Introducción

  1. Familiarización con el entorno Parrot OS y sus herramientas integradas.
  2. Estudio y uso del material didáctico: PDF LPI-01 en español para la certificación Linux.
  3. Definición de metas semanales para cubrir los temas principales.

Gestión de usuarios y permisos

  1. Creación y administración de usuarios y grupos.
  2. Configuración de permisos en archivos y directorios.
  3. Implementación de políticas de seguridad básicas para usuarios.

Gestión de paquetes

  1. Uso de APT para instalación, actualización y eliminación de paquetes.
  2. Configuración de repositorios y resolución de dependencias.
  3. Gestión de paquetes desde la línea de comandos y archivos .deb.

Administración del sistema

  1. Gestión de procesos y servicios en ejecución.
  2. Monitorización del sistema con herramientas como top, htop, y free.
  3. Programación de tareas automáticas con cron y at.
  4. Optimización del rendimiento del sistema.

Almacenamiento y sistemas de archivos

  1. Particionado y formateo de discos con herramientas como fdisk y mkfs.
  2. Montaje y desmontaje de sistemas de archivos.
  3. Gestión de puntos de montaje en el archivo fstab.
  4. Configuración y uso de discos externos cifrados.

Seguridad básica

  1. Configuración de firewalls con UFW y herramientas avanzadas como iptables.
  2. Implementación de autenticación SSH segura.
  3. Revisión de logs del sistema con journalctl y syslog.
  4. Análisis básico de vulnerabilidades y solución de problemas.

Redes y conectividad

  1. Configuración de interfaces de red y resolución de problemas de conectividad.
  2. Gestión de conexiones VPN y configuración de túneles seguros.
  3. Uso de herramientas como ping, traceroute y netstat para diagnóstico de red.

Pruebas y simulaciones

  1. Resolución de ejercicios prácticos del PDF LPI-01.
  2. Simulación de entornos para administración de servidores Linux.
  3. Práctica con configuraciones de usuarios, permisos y servicios en escenarios reales.

Evaluación y retroalimentación

  1. Autoevaluaciones basadas en los temas cubiertos.
  2. Identificación de áreas de mejora y ajuste del plan según necesidades.
  3. Preparación para la certificación Linux Essentials.
