Práctica, Proyecto base de datos Reserva Vuelos

Este curso es propiedad de incanatoit, solo se utiliza como medio de aprendizaje y como una excelente alternativa para aprender gestión de bases de datos.

    •                         -- CREACIÓN DE LA BASE DE DATOS ReservaBoletos -- 04/12/2024 
                        
    En esta línea estamos creando la base de datos llamada ReservaBoletos.
    
                    
                        CREATE DATABASE ReservaBoletos
                        ON PRIMARY(
                            NAME = 'ReservaBoletosDat',
                            FILENAME = 'C:\BasesDatosSQL\ReservaVuelos\ReservaBoletosDat.mdf',
                            SIZE = 10MB,
                            MAXSIZE = 40MB,
                            FILEGROWTH = 5MB
                        ),  

                        
    Definimos un segundo archivo de datos para distribuir mejor la carga de almacenamiento.
    
                    
                        (
                            NAME = 'ReservaBoletosMDF',
                            FILENAME = 'C:\BasesDatosSQL\ReservaVuelos\ReservaBoletosMDF.mdf',
                            SIZE = 5MB,
                            MAXSIZE = 30MB,
                            FILEGROWTH = 5%
                        )

                        
    Esta sección configura el archivo de LOG (transacciones):
    
                    
                        LOG ON (
                            NAME = 'ReservaBoletosLog',
                            FILENAME = 'C:\BasesDatosSQL\ReservaVuelos\ReservaBoletosLog.Ldf',
                            SIZE = 4MB,
                            MAXSIZE = 20MB,
                            FILEGROWTH = 10%
                        );

    Explicacion del codigo:

    Aquí definimos el archivo de datos principal (.mdf), con:

    • NAME: Nombre lógico del archivo.
    • FILENAME: Ruta donde se almacenará el archivo físico.
    • SIZE: Tamaño inicial del archivo.
    • MAXSIZE: Tamaño máximo que puede alcanzar.
    • FILEGROWTH: Tamaño en que crecerá cuando se llene.

    Esta sección configura el archivo de LOG (transacciones):

    • NAME: Nombre lógico del archivo de log.
    • FILENAME: Ubicación del archivo físico.
    • SIZE: Tamaño inicial.
    • MAXSIZE: Tamaño máximo permitido.
    • FILEGROWTH: Incremento de tamaño cuando se llena.
