    Explicacion del codigo:

    1. Tabla: Pais

    • PaisID: Identificador único del país.
    • Nombre: Nombre del país (único, no se repite).
    • Se establece PK_PaisID como clave primaria.

    2. Tabla: Pasajero

    • PasajeroID: Identificador único del pasajero.
    • Nombre, Apaterno, Amaterno: Datos personales del pasajero.
    • Tipo_documento y Num_documento: Información del documento de identidad.
    • Fecha_nacimiento: Fecha de nacimiento.
    • PaisID: Relación con la tabla Pais.
    • Telefono y Email: Datos de contacto (email debe ser único).
    • Clave: Contraseña del pasajero.
    • Se establece PK_Pasajero como clave primaria.
    • Se crea la FK_PasajeroPaisID que referencia a PaisID.

    3. Tabla: Aeropuerto

    • AeropuertoID: Identificador único del aeropuerto.
    • Nombre: Nombre del aeropuerto.
    • PaisID: Relación con la tabla Pais.
    • Se establece PK_AeropuertoID como clave primaria.
    • Se crea la FK_AeropuertoPaisID que referencia a PaisID.

    4. Tabla: Aerolinea

    • AerolineaID: Identificador único de la aerolínea.
    • Ruc: Registro único de contribuyente (único).
    • Nombre: Nombre de la aerolínea (único).
    • Se establece PK_AerolineaID como clave primaria.

    5. Tabla: Avion

    • AvionID: Identificador único del avión.
    • AerolineaID: Relación con la tabla Aerolinea.
    • Fabricante, Tipo: Información del avión.
    • Capacidad: Número de asientos disponibles.
    • Se establece PK_AvionID como clave primaria.
    • Se crea la FK_AvionAerolinea que referencia a AerolineaID.

    6. Tabla: Asiento

    • AsientoID: Identificador único del asiento.
    • Letra y Fila: Ubicación del asiento dentro del avión.
    • Se establece PK_AsientoID como clave primaria.

    7. Tabla: Tarifa

    • TarifaID: Identificador único de la tarifa.
    • Clase: Tipo de clase del boleto (Económica, Business, etc.).
    • Precio e Impuesto: Costo del boleto y su impuesto.
    • Se establece PK_Tarifa como clave primaria.

    8. Tabla: Reserva

    • ReservaID: Identificador único de la reserva.
    • Costo: Precio total de la reserva.
    • Fecha: Fecha en que se hizo la reserva (por defecto, la fecha actual).
    • Observacion: Información adicional.
    • Se establece PK_ReservaID como clave primaria.

    9. Tabla: Vuelo

    • VueloID: Identificador único del vuelo (autoincrementable).
    • AsientoID, AeropuertoID, ReservaID, AvionID, TarifaID: Relaciona el vuelo con otras tablas.
    • Se establece PK_VueloID como clave primaria.
    • Se crean las siguientes claves foráneas:
      • FK_VueloAsientoID → AsientoID
      • FK_VueloAeropuertoID → AeropuertoID
      • FK_VueloReservaID → ReservaID
      • FK_VueloAvionID → AvionID
      • FK_VueloTarifaID → TarifaID

    10. Tabla: Pago

    • PagoID: Identificador único del pago.
    • ReservaID: Relación con la tabla Reserva.
    • PasajeroID: Relación con la tabla Pasajero.
    • Fecha: Fecha del pago (por defecto, la fecha actual).
    • Monto: Cantidad pagada.
    • Tipo_Comprobante y Num_Comprobante: Información del comprobante del pago.
    • Impuesto: Valor del impuesto.
    • Se establece PK_PagoID como clave primaria.
    • Se crean las siguientes claves foráneas:
      • FK_ReservaID → ReservaID
      • FK_PasajeroID → PasajeroID
    • Se agrega la restricción CHK_Pago_Fecha para asegurar que la fecha del pago no sea mayor a la actual.
    • CHAR(n) → Cadena de texto de longitud fija de n caracteres.
      • Ejemplo: CHAR(4) almacena exactamente 4 caracteres, útil para códigos de países.
    • NVARCHAR(n) → Cadena de texto de longitud variable de hasta n caracteres.
      • Ejemplo: NVARCHAR(30) permite almacenar hasta 30 caracteres, ideal para nombres.
    • INT → Número entero sin decimales.
      • Ejemplo: INT para identificadores únicos como AerolineaID.
    • INT IDENTITY(1,1) → Número entero autoincremental.
      • Ejemplo: VueloID INT IDENTITY(1,1) genera valores secuenciales desde 1.
    • MONEY → Representa valores monetarios con hasta 4 decimales.
      • Ejemplo: Precio MONEY para almacenar costos de boletos.
    • DECIMAL(p,s) → Número decimal con precisión p y s decimales.
      • Ejemplo: DECIMAL(5,2) almacena números hasta 999.99, útil para impuestos.
    • DATE → Almacena una fecha en formato YYYY-MM-DD.
      • Ejemplo: Fecha_nacimiento DATE para registrar cumpleaños.
    • DEFAULT GETDATE() → Asigna automáticamente la fecha y hora actual.
      • Ejemplo: Fecha DATE DEFAULT GETDATE() registra la fecha de una reserva al insertarse.
